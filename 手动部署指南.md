## 方式一：通过微信开发者工具部署（推荐）

### 第1步：打开项目

1. 打开微信开发者工具
2. 点击「导入项目」
3. 填写：
   - **项目目录**: 选择 `/home/ubuntu/weapp-sober/` 目录
   - **AppID**: `wxccc054cbe7251344`
4. 点击「导入」

### 第2步：确认云环境

1. 在开发者工具顶部工具栏，点击「云开发」按钮
2. 确认右上角显示的环境为 `cloud1-d8gshwhb5dc72723e`（如不是，请切换）

### 第3步：上传云函数

**方法 A - 逐个上传（推荐）：**

在开发者工具左侧的「云开发」->「云函数」中，对每个函数点击右键 -> 「上传并部署」：

1. 右键 `login` → 「上传并部署：云端安装依赖」
2. 右键 `habits` → 「上传并部署：云端安装依赖」
3. 右键 `checkin` → 「上传并部署：云端安装依赖」
4. 右键 `stats` → 「上传并部署：云端安装依赖」

> ⚠️ 务必选择「云端安装依赖」，不要选「不上传 node_modules」。虽然本地已装好 node_modules，但云端环境需要重新安装以保证兼容性。

**方法 B - 批量上传：**

在 `cloudfunctions` 目录上右键 → 「上传并部署所有云函数」→ 选择「云端安装依赖」

### 第4步：创建数据库集合

1. 在云开发控制台中，点击「数据库」
2. 创建以下集合：

#### 集合1：`habits`

- **集合名**: `habits`
- **权限设置**（安全规则）:
  ```json
  {
    "read": "doc.openid == auth.openid",
    "write": "doc.openid == auth.openid"
  }
  ```

#### 集合2：`checkins`

- **集合名**: `checkins`
- **权限设置**（安全规则）:
  ```json
  {
    "read": "doc.openid == auth.openid",
    "write": "doc.openid == auth.openid"
  }
  ```

> 上述规则确保用户只能读写自己的数据。

### 第5步：验证部署

1. 在开发者工具的「云开发」控制台中，检查各云函数状态是否为「已部署」
2. 在项目开发环境中按 `F5` 刷新，查看控制台是否有报错
3. 或在模拟器中操作小程序，测试注册/打卡功能

## 常见问题

**Q: 上传时提示「未找到云环境」**
A: 确保 AppID 正确，且已开通云开发并创建了 `cloud1-d8gshwhb5dc72723e` 环境

**Q: 云函数调用返回 `errCode: -404011` 错误**
A: 云函数尚未部署或部署失败，请重新上传

**Q: 数据库读写报权限错误**
A: 检查数据库集合的安全规则是否正确配置（见第4步）

**Q: node_modules 是否需要上传？**
A: 不需要上传 node_modules，选择「云端安装依赖」即可
